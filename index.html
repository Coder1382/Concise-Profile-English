REST API app implements One-to-Many and Many-toMany relations as follows:
  • show one/all table record(s) (GET request with/without parameter)
  • add a table record (POST request)
  • update a table record (PUT request)
  • delete a table record (DELETE request)
There are three tables at hand: Fruit, Sellers. Suppliers. A record in the Fruit table has two mandatory fields: Name and Price. Each record has a list of sellers. User can update the price, and add/remove sellers to/from the list.
A record in the Sellers table has one mandatory field: Name, and one optional field: Supply. Each record has a list of fruit traded by a seller. User can change a supplier for a certain seller, and add/remove fruit to/from the list.
A record in the Suppliers table includes one mandatory field: Name. Each record has a list of sellers served by such a supplier. User can add/remove sellers to/from the list.
Upon adding, editing, removing records in tables, lists associated with them get automatically updated/deleted.
